## Skills Reference

Before executing, load the `obsidian-markdown` skill for best practices on frontmatter format, wikilink syntax, callout types, and task checkbox format. All generated markdown must follow these conventions.

---

Scan the entire vault — daily notes, project contexts, meeting notes, decision records.
Look for:

1. Problems I keep mentioning but haven't solved
2. Technologies or approaches I've noted but haven't explored
3. Connections between client work that could yield reusable tools
4. Gaps in my knowledge base (topics I reference but have no dedicated notes for)
5. Patterns in what energizes me vs what drains me (based on tone in notes)

Output the analysis directly in the conversation. Generate an ideas report with categories:
- **Tools to build** (based on repeated friction points)
- **People to connect with** (based on referenced contacts or domains)
- **Topics to investigate** (knowledge gaps)
- **Things to write** (ideas with enough supporting notes to become something)
- **Process improvements** (workflow patterns that could be automated)

For each idea, cite the source notes that led to it with [[wikilinks]].

## Follow-up

After presenting the ideas, offer to act on the top ones:
- "Want me to add any of these as tasks to your daily note? Pick by number."
- For investigation items: "Want me to create a note in /domains/[relevant]/ to start collecting thoughts on this?"
- For tool ideas: "Want me to scaffold a project for this?"

## Archival

Also save to `/agent-output/ideas/ideas-report-[date].md`.
